mgmt in a nutshell

  • spiritual successor to Puppet
  • operates on graphs (DAGs)
  • built in golang
  • can run without pre-installed agents
  • no central server ever
  • events, parallelism, node merging

Running mgmt from Puppet code

Essentially





{
  file { "/tmp/a": }
  ->
  file { "/tmp/b": }
}






resources:
  file:
  - name: /tmp/a
    path: /tmp/a
  - name: /tmp/b
    path: /tmp/b
edges:
- name: File[/tmp/a] -> File[/tmp/b]
  from:
    kind: file
    name: /tmp/a
  to:
    kind: file
    name: /tmp/b
